🎯 面试核心要点

一、项目介绍（1分钟电梯演讲）

标准模板（背下来）

这是一个平台化代码治理工具，解决历史代码冗余问题。


背景：鲁班内控平台经过3年迭代，累积200+零流量接口和3000+行无用代码，人工治理耗时2小时/接口，误删风险高。


方案：我们打通"FAST流量监控 → 字节码解析 → 级联删除 → 编译校验 → Git推送"全链路，采用java-all-call-graph做静态分析，基于DFS算法递归识别可删除方法，配合黑名单保护+编译校验确保安全。


成果：单接口治理从2小时降至15分钟，效率提升87.5%；3个月清理173个接口、18,827行代码，节省42.8天人力；误删率0%，线上故障0起；ROI达25.8%。


亮点：采用Vibe Coding（AI协同开发），开发效率提升60%，25天工作量压缩到10天。

二、高频面试问题

Q1: 为什么要做这个项目？解决了什么痛点？

回答要点：

  1. 业务背景：代码仓库3年累积200+零流量接口，影响编译速度和代码可读性
  2. 痛点1：人工治理耗时2小时/接口，无法规模化
  3. 痛点2：调用链复杂，手工排查易遗漏，误删风险约15%
  4. 痛点3：缺乏统一平台，各团队各自为战，无审计记录
  5. 价值：通过平台化工具，将治理效率提升87.5%，3个月节省42.8天人力
Q2: 介绍一下核心算法，为什么用DFS？

回答要点：

  1. 算法目标：从零流量接口出发，递归分析所有只被该删除链路调用的方法
  2. 核心思想：如果方法M的所有调用方都在"待删除集合"中，且不在黑名单，则M可删
  3. 为什么DFS：
    • 调用链通常较深（10-20层），DFS符合调用栈自然结构
    • 空间复杂度低，只需维护当前路径栈
    • 支持提前剪枝，遇到黑名单方法立即停止
  4. 复杂场景：循环依赖用visited集合避免死循环，共享依赖通过"调用方检查"保留
Q3: 如何保证不误删？有哪些安全机制？

五重保障（重点）：

  1. 调用链分析：只删除"仅被删除链路调用"的方法，避免误删正在使用的代码
  2. 黑名单机制：保护@Scheduled、@KafkaListener等框架方法，支付类、核心链路自动保护（42个方法被保护）
  3. 编译校验：删除后立即Maven编译，失败自动回滚（git reset --hard），已拦截3次误删
  4. 测试流水线：自动触发156个单元测试+42个集成测试，全部通过才允许合并
  5. 人工审批：生成预览报告，Tech Leader审查后才执行删除

结果：上线后3个月，误删率0%，线上故障0起

Q4: 技术选型为什么选java-all-call-graph？

选型对比：

  1. java-all-call-graph：成熟开源工具，专注Java方法调用关系分析
  2. 优势：
    • 内置H2数据库，查询性能优秀（P99 < 200ms）
    • 支持泛型、Lambda、内部类等复杂场景
    • 社区活跃，文档完善，学习成本低
  3. 对比其他方案：
    • 自研引擎：成本高（需3-4周开发），场景覆盖不如成熟工具
    • 简单正则匹配：无法处理反射、动态代理等复杂场景
Q5: Vibe Coding是什么？具体怎么做的？

回答要点：

  1. 定义：AI协同开发范式，人负责核心算法和架构，AI辅助样板代码生成
  2. 分工：
    • 核心DFS算法、黑名单判定逻辑：100%人工
    • CRUD、Mapper接口、DO实体类：90%由AI生成
  3. 流程：我给AI写详细Prompt（包含表结构、查询需求、代码规范），AI生成后我人工校验SQL、补充异常处理
  4. 效果：25天工作量压缩到10天，开发效率提升60%，代码注释覆盖率从60%提升到85%
  5. 关键：AI是助手不是决策者，核心代码必须人工Review
Q6: 项目有哪些难点？如何解决的？

三大难点：

  1. 难点1：方法签名匹配
    • 问题：方法重载、泛型擦除、Lambda表达式
    • 解决：基于完整签名哈希（className:methodName(paramTypes):returnType）精确匹配
  2. 难点2：循环依赖处理
    • 问题：A→B→C→B会导致DFS死循环
    • 解决：visited集合记录已访问节点，检测到重复立即return
  3. 难点3：反射/动态代理识别
    • 问题：静态分析无法识别反射调用
    • 解决：黑名单机制保护@Resource、@Autowired等注解方法，配合编译校验兜底
Q7: 如果再做一次，有哪些可以改进的地方？

改进方向：

  1. 动态调用识别：结合运行时字节码增强（如JavaAgent），识别反射调用，降低漏报率
  2. 跨语言支持：扩展到Kotlin、Scala项目，服务更多团队
  3. 智能风险评估：基于历史变更记录、告警数据，自动评估方法删除风险
  4. 数据可视化：治理效果大盘、代码健康度趋势图，提升平台价值感知
